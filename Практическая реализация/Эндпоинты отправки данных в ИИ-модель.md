LM Studio предоставляет эти эндпоинты как OpenAI-совместимые, что позволяет использовать стандартные OpenAI-клиенты, просто изменив `base_url` на адрес вашего локального сервера (например, `http://localhost:1234/v1`).

Подробнее о каждом из них:

### `/v1/models` (GET)

Этот эндпоинт предназначен для получения списка моделей, доступных на вашем сервере LM Studio.

*   **Основное назначение**: Получить идентификаторы (`id`) моделей, которые можно использовать в других запросах.
*   **Плюсы**:
    *   Позволяет программно узнать, какие модели загружены или доступны для загрузки.
    *   Полезен для динамической настройки вашего агента.
*   **Минусы**:
    *   Это служебный эндпоинт, он не используется для генерации текста.

---

### `/v1/completions` (POST) - **Legacy (Устаревший)**

Это "классический" эндпоинт для текстовой генерации, где вы отправляете один текстовый промпт и получаете его продолжение.

*   **Основное назначение**: Генерация текста на основе простого промпта.
*   **Плюсы**:
    *   Простой и прямой. Дает вам полный контроль над формированием промпта.
    *   Подходит для базовых ("сырых") моделей, которые не были дообучены для ведения диалога.
*   **Минусы**:
    *   **Устаревший**: OpenAI больше не поддерживает этот эндпоинт, и его использование не рекомендуется для новых проектов.
    *   **Не применяет шаблон чата**: Для чат-моделей (как большинство современных) это критично. Без правильного шаблона промпта модель может генерировать бессвязный или неожиданный ответ.

---

### `/v1/chat/completions` (POST) - **Рекомендуемый для чатов**

Это основной и рекомендуемый эндпоинт для создания чат-интерфейсов. Он принимает структурированную историю сообщений (`messages`).

*   **Основное назначение**: Ведение диалога с моделью.
*   **Плюсы**:
    *   **Структурированный формат**: Использует массив сообщений с ролями (`system`, `user`, `assistant`), что идеально для управления историей диалога.
    *   **Автоматическое применение шаблона**: LM Studio автоматически применяет правильный шаблон чата для загруженной модели, что гарантирует корректную работу.
    *   **Современный стандарт**: Это основной эндпоинт, который используется в OpenAI API и большинстве современных приложений.
    *   **Изображения**: Поддерживает работу с картинками.
*   **Минусы**:
    *   Менее гибок, чем `completions`, если вы хотите полностью самостоятельно формировать промпт без структуры сообщений.

---

### `/v1/responses` (POST) - **Новейший и наиболее мощный**

Это новейший эндпоинт, реализующий OpenAI Responses API. Он позиционируется как более мощная замена `chat/completions`, особенно для агентов.

*   **Основное назначение**: Создание "ответов" с поддержкой продвинутых функций, таких как состояние, инструменты и рассуждения.
*   **Плюсы**:
    *   **Stateful (С сохранением состояния)**: Позволяет передавать `previous_response_id`. Это значит, что сервер LM Studio может сам управлять историей диалога, избавляя вас от необходимости хранить и отправлять её целиком при каждом запросе.
    *   **Поддержка инструментов (Tool Calling)**: Встроенная поддержка вызова функций, аналогично `chat/completions`.
    *   **Поддержка MCP**: Возможность использования удаленных MCP-серверов как инструментов (требуется опция в настройках).
    *   **Поддержка "рассуждений" (Reasoning)**: Для некоторых моделей можно контролировать усилия на "рассуждение" (`reasoning: { effort: "low" | "medium" | "high" }`).
*   **Минусы**:
    *   **Новизна**: Это относительно новый эндпоинт, поэтому экосистема вокруг него может быть менее зрелой, чем у `chat/completions`.
    *   **Сложность**: Предоставляет больше возможностей, но и требует более глубокого понимания для эффективного использования.

---

### `/v1/embeddings` (POST)

Этот эндпоинт предназначен не для генерации текста, а для создания векторных представлений (эмбеддингов) вашего текста.

*   **Основное назначение**: Преобразование текста в числовой вектор, который можно использовать для поиска по сходству, классификации или в системах RAG (Retrieval-Augmented Generation).
*   **Плюсы**:
    *   Незаменим для создания систем долговременной памяти и семантического поиска по вашей базе знаний.
*   **Минусы**:
    *   Не генерирует текст, а только векторы. Это инструмент для другой задачи.
