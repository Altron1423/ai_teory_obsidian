## Этапы обработки
### 1) Обработка данных
Для поддержания контекста будет использоваться [[Формовщик]]. Он составит контекст на основе истории общения ([[Историк]]), векторизованной базы знаний ([[RAG-система]]), графа смежных понятий, дополнительных источников (будут введены и уточнены позже)
### 2) Передача в модель
^523a62

Для передачи в модель будет использоваться [[Эндпоинт для модели (chat completions)]].
Структура сообщения для передачи в `messages`:
```json
[
	{
		"role": "system",
		"content": ... # Текст основного промпта, включающий в себя:
					   # Характер, правила поведения и использования инструментов
	},
	{
		"role": "user",
		"content": "Контекст: {...}" # Контектс составленный Формовщиком
	},
	{
		"role": "user",
		"content": [
			{
				"type": "text",
				"text": "Отправитель: {...}\nСообщение: {...}"
				# Отправителя, для понниманияя моделью, кто сейчас высказался.
				# Текст сообщения, если есть. Иначе только отправитель.
			},
			{
				"type": "image_url",
				"image_url": {
					"url": "data:image/png;base64,{...}"
				}
			}
			# Блок с картинкой существует только в случае существования картинки.
		]
	}
]
```

Вместе с этим в модель передаётся набор инструментов, доступных для вызова.
и параметр `tool_choice` со значением `"auto"`.

### 3) Получение ответа
На выход из модели ожидается одно из двух:
- вызов инструмента
  Стандартная структура вызова инструмента для работы.
- обычный текст, интерпретируемый как прямой ответ. Должен иметь дополнение в своей структуре:
  ```
  {...}
  [Кому: {...}, Куда: {...}]
  {...}
  ```
  **Пример заполнения**
  ```
  Ваня проснулся и появился в сети.
  [Кому: Ваня, Куда: личное]
  Доброе утро! Как дела?
  ```
   * Первыми идут внутренние мысли модели, они не будут отправлены, какому либо пользователю и останутся внутри модели для использования при формировании контекста и личного характера. Может быть единстввенным знче
   * Далее поле адресата:
    * `Кому` указывает получателя, который является целью ответа.
    * `Куда` указывает место: личка, группа или другой источник. Данный параметр не обязателен, при его отсутствии сообщение будет переслано в место получения.
   * Последним идёт текст сообщения в свободной форме. Должен существовать, если есть поле адресата.
  Отсутствия разделителя с адресатом говорит о том, что весь текст - внутренние мысли.

---
### 4) Оценка ответа
Вызов инструмента -> происходит вызов с помощью [[Продюсер|Продюсера]], поле чего запрос вызова, результат переходит в [[Формовщик]], который формирует новый контекст для работы. (возвращение на [[#^523a62"|2 пункт]])
Возврат текста -> происходит его обработка и возвращение ([[#^c1bcde|пункт 5]])

---
### 5) Обработка ответа
^c1bcde

За обработку отвечает [[Посыльный]]. Он получает входное сообщение, контекст, и ответ нейросети. Исходя из этого, он принимает решение, куда его передать: отправить в мессенджер, озвучить, сохранить проанализированную информацию и т.д.. Для достижения этих целей он так же может использовать [[Продюсер|Продюсера]], для вызова инструментов. Но в отличии от модели обладает своим набором инструментов.
При является важной частью при общении через мессенджер. Он анализирует имя, к которому обратилась модель, и место (личка, группа, …), изначальный запрос, контекст базы знаний и исходя из этого, выбирает место ответа, куда будет переслан запрос
